# 风险判定规则参考

## Python 高风险变更模式

| 模式 | 风险 | 说明 |
|------|------|------|
| `is` → `==` | 极高 | None/False/0 等 falsy 值可能绕过验证 |
| `is not` → `!=` | 极高 | 同上 |
| 返回 `Optional[T]` → 返回 `T` | 高 | 调用方可能依赖 None 判断 |
| 返回 `T` → 返回 `Optional[T]` | 高 | 调用方未做空值处理 |
| 删除 `try/except` | 高 | 异常向上传播 |
| `except Exception` → `except ValueError` | 高 | 缩小捕获范围 |
| `async def` → `def` | 中 | 可能阻塞 event loop |
| 超时值 × 10 | 中 | 级联等待风险 |
| `__init__` 参数变更 | 中 | 所有实例化点受影响 |
| 加密/哈希算法变更 | 高 | 兼容性断裂 |
| 重试次数减少 | 中 | 降低容错能力 |

## 调用类型与耦合度

| 调用类型 | 耦合度 | 风险传播衰减 |
|---------|--------|-------------|
| Python import 直接调用 | 强 | 0.8 |
| HTTP API 调用 | 中 | 0.6 |
| MQ 消息事件 | 弱 | 0.4 |
| 配置文件引用 | 最弱 | 0.3 |
| 动态分派 (getattr) | 不确定 | 0.5 |

## 测试覆盖判定

| 标注 | 条件 |
|------|------|
| ✅ 有测试 | coverage > 80% 或对应 test_*.py 存在且有该函数的测试 |
| ⚠️ 覆盖不全 | coverage 40-80% 或测试存在但缺少边界 case |
| ❌ 无测试 | 无对应测试文件或 coverage < 40% |
